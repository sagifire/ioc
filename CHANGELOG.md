# Changelog

The root workspace is private and is not published.

## 0.0.3 release handoff

Publishable package versions and package changelogs were fixed at `0.0.3` after the Stage
18 graph export, async multi-provider, lifetime validation and final schema-evolution
stabilization tasks passed their audits and human review.

This repository state prepares the synchronized `@sagifire/ioc`, `@sagifire/ioc-next` and
`@sagifire/ioc-testing` package set for the controlled GitHub Actions release path. Version
fixation alone does not prove npm publication; registry and provenance evidence are recorded
separately by the release task.

## 0.0.2 release handoff

Publishable package versions and package changelogs were fixed at `0.0.2` after Stage 17
implementation, full audit, stabilization handoff and release-preparation run.

This repository state prepares the synchronized `@sagifire/ioc`, `@sagifire/ioc-next` and
`@sagifire/ioc-testing` package set for the human-controlled release path. Repository files
and version fixation alone do not prove npm publication.

Actual npm publishing remains gated by explicit human approval, reviewed workflow dispatch
and external GitHub/npm settings.

## 0.0.1 stabilization handoff

Publishable package versions and package changelogs were fixed at `0.0.1` after the Stage
16 stabilization audit and review-approved closure of pre-release audit blockers.

No npm publish has been performed from this repository. Publishing remains gated by
explicit human approval, reviewed workflow dispatch and external GitHub/npm settings.

Package changelogs are generated by Changesets when `pnpm changeset:version` is run:

- `packages/ioc/CHANGELOG.md`
- `packages/ioc-next/CHANGELOG.md`
- `packages/ioc-testing/CHANGELOG.md`

Release note drafts are stored as human-reviewable files in `.changeset/` before version
and changelog updates are applied.

Release validation and manual npm publishing are documented in
[`docs/release.md`](docs/release.md).
