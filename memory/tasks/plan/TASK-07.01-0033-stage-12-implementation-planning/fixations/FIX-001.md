## Stage 12 Implementation Plan

Stage 12 split into five sequential implementation tasks.

### TASK-07.01-0034: Testing package foundation and isolated test runtime

Purpose:

- replace the placeholder package surface with the first public testing helpers;
- implement an isolated test runtime factory over `createContainer()` / `freeze()`;
- keep each test runtime created from fresh configuration;
- keep `@sagifire/ioc-testing` as a package that depends on `@sagifire/ioc`, not the
  other way around;
- leave composer overrides, module harnesses and graph assertions for later tasks.

### TASK-07.01-0035: Overrides and test composer

Purpose:

- implement explicit override declarations for test configuration;
- implement a test composer helper over existing `createComposer()` semantics;
- apply overrides before `compose()`;
- support required-port binding overrides without mutating composed runtime;
- preserve diagnostics and graph visibility through existing composer APIs.

### TASK-07.01-0036: Module harness and fake modules

Purpose:

- implement fake module helpers for explicit capabilities and setup behavior;
- implement a module harness for testing one module with fake required ports and support
  modules;
- preserve module-private provider isolation and required-port ownership;
- keep fake modules as explicit module definitions, not auto-discovered fixtures.

### TASK-07.01-0037: Graph and diagnostic assertions

Purpose:

- implement graph assertion helpers over public `ModuleGraph`, `ComposerInspection` and
  `RuntimeInspection` data;
- implement diagnostic assertion helpers over public `DiagnosticReport` data;
- produce deterministic readable assertion errors usable from Vitest;
- avoid reading private provider values or internal runtime structures.

### TASK-07.01-0038: Testing package hardening, exports and docs

Purpose:

- harden type inference and runtime coverage across the final testing helper surface;
- verify root package export smoke tests for `@sagifire/ioc-testing`;
- update README/docs/testing documentation enough to explain testing workflows;
- verify package boundary: core does not import testing package and testing package does
  not include Next.js adapters;
- keep Stage 13 Next.js helpers out of scope.

## Stage 12 Decisions

- `@sagifire/ioc-testing` is the only package that owns testing helpers.
- `@sagifire/ioc-testing` may depend on `@sagifire/ioc`; `@sagifire/ioc` must not depend on
  `@sagifire/ioc-testing`.
- Testing helpers create fresh builders/composers/runtimes; they never mutate a frozen
  production runtime.
- Overrides are explicit token-level declarations and are applied before `freeze()` or
  `compose()`.
- Composer/test overrides should satisfy required ports or test runtime configuration;
  replacing public capabilities should use fake modules or an explicit test module.
- Fake modules are explicit module definitions and must remain inspectable through the
  existing graph APIs.
- Module harnesses test one module under controlled fake ports/modules while preserving
  module-private provider isolation.
- Graph assertions operate on public inspection data only.
- Diagnostic assertions operate on public `DiagnosticReport` / typed error data only.
- Assertion helpers should produce deterministic plain errors readable in Vitest without
  requiring hidden access to Vitest internals.
- Stage 12 does not implement Next.js adapters, route/action scopes, documentation examples
  beyond minimal testing docs, release automation or core runtime semantic changes.
