## Rationale

Roadmap і state мають відображати approved implementation order та shared-foundation
gate. Без цього async і lifetime backlog можуть активуватися за різними identity/inspection
assumptions або створити паралельні provider models.

## Impact and risks

- Canonical changes описують approved plan, але не implemented behavior.
- Derived task packages і prepared async task reconciliation мають бути виконані в тій
  самій finalization операції після окремого approval follow-up.
- Якщо follow-up task creation відхилено, цей FIX не застосовується.
- Version, changelog, publish workflow і release status не змінюються.
