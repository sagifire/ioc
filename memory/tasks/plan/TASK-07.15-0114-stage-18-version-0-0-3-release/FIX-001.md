## Причина

Canonical Project Memory досі описує `0.0.2` як останню release task і вимагає окремий
майбутній `0.0.3` handoff. RUN-001 уже довів readiness, зафіксував manifests/changelogs at
`0.0.3` і пройшов local release validation, але зовнішній release ще не виконано. Щоб не
створювати неправдивий publish claim, цю фіксацію можна застосувати лише після task-level
human approval, successful GitHub `dry-run` і `publish` workflows та post-publish registry /
provenance verification для всіх трьох packages.

## Application condition

Apply only after all of these facts exist on one exact commit: task-level approval, remote
`master` SHA parity, successful manual `dry-run`, successful manual `publish`, registry
`0.0.3` + `latest` for all three packages and available workflow provenance evidence. If any
fact is absent or registry state is partial, do not apply this fixation and block finalization.

## Impact

- Product roadmap and current state become consistent with the externally verified release.

  do not change.
