## Stage 13 Implementation Plan

Stage 13 split into five sequential implementation tasks.

### TASK-07.01-0040: Next package foundation and cached runtime helper

Purpose:

- replace the placeholder package surface with the first public Next adapter helpers;
- implement `createNextRuntime()` or equivalent cached runtime factory over existing
  `@sagifire/ioc` runtime/composer/app APIs;
- define cache identity and failure retry behavior explicitly;
- keep runtime caching outside core and avoid hidden global mutable container semantics;
- leave request context, route helpers and server action helpers for later tasks.

### TASK-07.01-0041: Request context and scoped values

Purpose:

- implement `createNextRequestContext()` or equivalent helper for explicit request or
  operation data;
- convert request context entries to existing `CreateScopeOptions` / scope-local values;
- preserve token-typed inference for request-specific values;
- keep context data explicit and serializable where practical;
- leave route/action wrapper helpers for later tasks.

### TASK-07.01-0042: Route handler scope helper

Purpose:

- implement route handler helper over cached runtime and request context;
- create one scope per route invocation;
- pass scope/runtime/context explicitly to handler callback;
- dispose the scope on success and failure;
- avoid requiring business logic to live in route handlers.

### TASK-07.01-0043: Server action scope helper

Purpose:

- implement server action helper over cached runtime and operation context;
- create one scope per server action invocation;
- support async action callbacks and disposal on success/failure;
- keep server action helper independent from route-specific request/response behavior;
- preserve scope-local values for action-specific inputs.

### TASK-07.01-0044: Next examples, hardening, exports and docs

Purpose:

- harden type inference and runtime coverage across final `@sagifire/ioc-next` helper
  surface;
- verify package exports for `@sagifire/ioc-next`;
- add minimal App Router examples/docs for runtime caching, route handler scope and server
  action scope;
- verify core package has no Next.js/React imports;
- keep broader Stage 14 documentation/examples and Stage 15 release automation out of
  scope.

## Stage 13 Decisions

- `@sagifire/ioc-next` is the only package that owns Next.js adapter helpers.
- `@sagifire/ioc-next` may depend on `@sagifire/ioc`; `@sagifire/ioc` must not depend on
  `@sagifire/ioc-next`, Next.js or React.
- Runtime caching belongs at adapter/application boundary and must not create a core
  global container or service locator.
- Cached runtime initialization should deduplicate in-flight creation; failed
  initialization should not permanently poison cache unless an implementation task
  documents a stronger policy.
- Request, route and action helpers create scopes through existing core runtime APIs and
  dispose them after callback execution.
- Request context is explicit scope-local data; helpers must not use hidden async-local
  context unless a later task records a specific decision.
- Route handler helper and server action helper pass runtime/scope/context explicitly to
  user callbacks.
- Stage 13 does not implement filesystem auto-discovery, route scanning, module discovery,
  decorators, `reflect-metadata`, release automation or broad Stage 14 docs.
- Next.js/React dependencies should be peer or optional type-only dependencies only if the
  implementation task proves they are needed for public types or examples.
