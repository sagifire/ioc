## Stage 4 Implementation Plan

Stage 4 implementation task має змінювати тільки core package container surface:

- `packages/ioc/src/container.ts`
- `packages/ioc/src/index.ts`, якщо потрібен root re-export
- tests для sync provider runtime behavior і public type inference
- package export smoke tests, якщо потрібно оновити expectations для `@sagifire/ioc/container`
- docs/README тільки там, де Stage 4 public API description інакше буде misleading

Planned public API baseline:

- `createContainer()`.
- `ContainerBuilder` mutable configuration API.
- `ContainerRuntime` immutable runtime API.
- `ResolutionContext` для sync factories with `get()` і `tryGet()`.
- `bind(token).toValue(value)`.
- `bind(token).toFactory(factory)`.
- `bind(token).toClass(ClassConstructor)`.
- `.singleton()` і `.transient()` lifetime modifiers для factory/class providers.
- `runtime.get(token)`.
- `runtime.tryGet(token)`.

Planned Stage 4 decisions:

- `freeze()` returns `Promise<ContainerRuntime>` even though Stage 4 has only sync
  providers.
- `toValue` is singleton by definition.
- `toFactory` is transient by default.
- `toClass` is transient by default.
- `toClass()` accepts no-argument constructors only (`new () => TValue`).
- Classes with dependencies must be wired explicitly through `toFactory(({ get }) => ...)`.
- Token ID is provider key; token object identity is not used for registration matching.
- `tryGet()` returns `undefined` only for missing providers and still throws provider
  cycles or provider execution errors.
- Provider cycles are detected on resolution stack and report a readable token ID path.
- Duplicate single-provider token registration fails before runtime resolution.

Planned minimal errors:

- `ProviderNotFoundError` or equivalent for missing `get()`.
- `DuplicateProviderError` or equivalent for duplicate `bind()`.
- `ProviderCycleError` or equivalent for provider-level cycles.
- `ContainerFrozenError` or equivalent for configuration mutation after freeze.

These errors may have stable local `code` strings and readable messages, but Stage 4 must
not introduce `SagifireIocError`, `DiagnosticReport` or diagnostic formatting.

## Rationale

Stage 4 is the point where tokens become executable dependency identity. The main risk is
building a container API that solves the immediate sync provider cases but blocks later
async/scoped/multi-provider stages. Keeping `freeze()` async-compatible avoids a breaking
API change in Stage 7. Keeping `toClass()` no-magic preserves the project rule against
decorators, `reflect-metadata`, constructor parameter metadata and hidden service-locator
behavior.

A single Stage 4 implementation task is preferable to splitting the stage now because
`toFactory()`, lifetimes and cycle detection are tightly coupled. A split would likely
create an interim public container surface with incomplete lifecycle semantics.
