# Domain Rules

1. Доменні поняття фіксуються в `glossary.md`.
2. Поточний прийнятий стан домену фіксується в `current/`.
3. Цільовий або бажаний стан фіксується в `target/`.
4. Відкриті питання фіксуються в `open-questions.md`.
5. Доменні правила не повинні суперечити product requirements.
6. Поточний стан (`current/`) і цільовий стан (`target/`) не змішуються.
7. Декомпозицію `current/` і `target/` визначає людина разом з Agent Assistant.

## Additional Context

Якщо правило ще не прийняте, воно має бути оформлене як питання або proposal, а не як факт.
